EagleData

Engineering applications over Social and Open Data through Domain-Specific Languages

From Angel Mora-Segura at March, 2016

Our goal is to facilitate the construction of applications using open data and social networks as communication platform. For this purpose, we propose a family of domain-specific languages directed to automate the different tasks involved, like describing the structure and semantics of the heterogeneous data sets, the patterns to be sought in social network messages, the information to be extracted from static and dynamic data and the messages (over social networks) that the system needs to produce. We have built an extensible working prototype, which allows adding new open data formats, and support for different social networks.

Parsing Cells / Describing nodes with values

Uniqueness Quality:
[KEY | UNIQUE]
Type:
[STRING | INTEGER | FLOATSTRING | DATE | TIME | YEAR | MONTH | DAY | BOOLEAN | LAT | LONG]
Name:
An unique identificator.
Name use in the representation:
Must use if representation have a specific name for the list of values, if not descriptions will be taken in order.
{key = value, ...} //OPTIONS
The list of options by type is presented below
TYPE OPTIONS EXAMPLES
INTEGER
  • NULL
  • SEPARATOR
  • DEFAULT
{NULL = "99"}
{SEPARATOR = " "}
This involves a value "1 5 7" will be processed as a list [1, 5, 7]
{DEFAULT = "3"}
Empty values will be processed as a "3" by default

  • REGEX

[error] STRING. REGEX is not valid"
{REGEX_FLAGS = "CASE_INSENSITIVE"}
Acording to the Regular expression flags admited by java.util.regex.
[error] REGEX_FLAGS can not be called without REGEX option"
[error] STRING. REGEX_FLAGS is not valid"
FLOATSTRING
  • NULL
  • SEPARATOR
  • DEFAULT
  • DECIMALCHAR
  • PATTERN
{NULL = "nil"}
{SEPARATOR = " "}
This involves a value "1 5.1 7.0" will be processed as a list [1, 5.1, 7.0]
{DEFAULT = "3"}
Empty values will be processed as a "3" by default
{DECIMALCHAR = "."}
7.0 will be OK but 7,0 will be not valid
[warning] DECIMALCHAR. You should consider to specify a character for the decimal separator."
[error] DECIMALCHAR. Option must be a ".", ",", "\'"
{PATTERN = "^\d+(\.\d+)?%"}
"6%", "7.5%" and "1.80%" are valid, if the value is not valid will be process as a STRING

	format = "YYYY/YY"
},

enumeration grade {"A", "B", "C", "D", "E", "F"},

description CSV classList {
	int classId {null = "99"},
	string name,
	date birthDate {format = "ddmmYYYY", separator = "/"},
	course firstEnrolled,
	grade finalGrade,
	string id {null = " ", regex = "\\d{8}[A-Za-z]"},
	boolean passExam {format = "1 | 0"}
}
