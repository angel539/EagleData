EagleData

Engineering applications over Social and Open Data through Domain-Specific Languages

From Angel Mora-Segura updated at June, 2016. September, 23rd at MEDI2016

Engineering applications over Social and Open Data through Domain-Specific Languages

 6th International Conference on Model and Data Engineering, 2016. Almería, Spain. Ángel Mora Segura, Juan de Lara download

Abstract

There is a current trend among governments and organizations to make all sort of information (like budgets, demographic or economic data) public. The information released in this way is called {\em Open Data}. Many institutions promote the creation of innovative applications using the data they have released, e.g., in combination with social networks, but only highly skilled engineers can accomplish this task.

Our goal is to facilitate the construction of applications using open data and social networks as communication platform. For this purpose, we propose a family of domain-specific languages directed to automate the different tasks involved, like describing the structure and semantics of the heterogeneous data sets, the patterns to be sought in social network messages, the information to be extracted from static and dynamic data and the messages (over social networks) that the system needs to produce. We have built an extensible working prototype, which allows adding new open data formats and support for different social networks.

#1_ Fragments

A fragment is a format-independent, reusable description for describing the semantics of data in heterogeneous format. 
@transport
package BusLine{
    datatype Id : Int (minLen = 1, maxLen = 3)
    fragment Line{
        key Id id
        String source 
        String destination 
        GeoPoint sourcePoint (null = "")
        GeoPoint destinationPoint (null = "")
        Stop[*] stops   
    }
    fragment Stop{
        key Int stopID
        String name
        GeoPoint stopPoint
    }
    fragment GeoPoint{
        Lat latit (min = -90.0, max = 90.0)
        Long longit (min = -180.0, max = 180.0)
    }
}
@city
@geo
package Address{
    enumeration AddrKind{"Calle","Avenida","Paseo"}
    datatype PostalCode : String {
        Digits[2] city 
        Digits[3] district 
    }
    fragment Address{
        AddressPoint addressPoint
        String others
        PostalCode postalcode
    }
    fragment AddressPoint{
        AddrKind kind
        String name
        Int number
    }
}
